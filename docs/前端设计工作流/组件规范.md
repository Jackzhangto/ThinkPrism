# 组件规范（Token 分层与组件映射）

> 与《[ThinkPrism 标准前端样式规范](./ThinkPrism%20标准前端样式规范.md)》原则一致。
> 本项目采用 **React + Tailwind CSS**，Design Tokens 落地为 `tailwind.config.ts` 配置。

---

## 一、总体设计原则

| 原则 | 说明 |
|------|------|
| **组件不写具体数值** | 组件永远不直接使用具体数值（margin/gap/font-size/color 等），必须使用 Tailwind Utility Class。 |
| **组件只认语义 Token** | 使用 `p-card` 而非 `p-[20px]`；使用 `text-text-primary` 而非 `text-slate-50`。 |
| **Layout Token ≠ 视觉 Token** | Popup 宽度、Page Padding 仅在页面根容器使用；普通组件禁止引用。 |
| **业务组件 ≠ 视觉组件** | `ui/` 组件负责样式；`business/` 组件负责逻辑组合，禁止在业务组件中写复杂 CSS。 |

---

## 二、Token 分层结构 (Tailwind Config)

所有的 Token 必须在 `tailwind.config.ts` 的 `theme.extend` 中定义。

```ts
// tailwind.config.ts 结构示意
export default {
  theme: {
    extend: {
      colors: { ... },      // 语义化颜色 (bg, text, border, brand)
      spacing: { ... },     // 语义化间距 (page, section, card, stack)
      fontSize: { ... },    // 语义化字体 (h1, body, hint)
      borderRadius: { ... },// 语义化圆角 (card, button)
      boxShadow: { ... },   // 语义化阴影 (card, popover)
    }
  }
}
```

---

## 三、Layout Tokens (页面级)

- **含义**：Popup 宽度、整页 Padding、区块间距。
- **使用规则**：只允许 **Popup App 根容器** 或 **Section 容器** 使用。

示例：
- `w-[400px]` (Popup 固定宽)
- `p-page` (整页内边距)
- `gap-section` (大区块间距)

---

## 四、Spacing Tokens

- **严禁**：`m-[14px]`, `gap-[18px]`。
- **正确**：`gap-stack`, `p-card`, `my-sm`。

---

## 五、Typography Tokens

- **规则**：组件不允许定义新字号；标题只用 `text-h1` / `text-h2`，正文只用 `text-body` / `text-sub`。
- **配置**：在 Tailwind 中配置 `fontSize` 为 `[fontSize, { lineHeight, fontWeight }]` 数组格式，一键应用。

---

## 六、Color Tokens

- **关键**：颜色按**用途**命名（`bg-panel`, `text-primary`, `border-DEFAULT`）。
- **禁止**：`text-blue-500`, `bg-gray-300` 等直接引用色板（除非用于临时调试）。

---

## 七、Radius / Shadow Tokens

- 圆角、阴影语义化。
- `rounded-card`, `rounded-button`。
- `shadow-card`, `shadow-popover`。

---

## 八、Token → 组件映射规范

| 组件类型 | 允许使用的 Tailwind 类 | 禁止 |
|----------|------------------|------|
| **布局（Container/Stack）** | `gap-*`, `p-*`, `bg-*`, `rounded-*` | `shadow-*` (除非是 Card), `text-*` (除非统一设置) |
| **文本（Text/Title）** | `text-h1`, `text-body`, `text-text-*`, `font-*` | `m-*` (由容器控制间距) |
| **Card** | `p-card`, `rounded-card`, `bg-panel`, `shadow-card`, `border` | 卡片嵌套最多 2 层（见下） |
| **Button** | `px-*`, `py-*`, `rounded-button`, `bg-brand`, `hover:bg-*` | — |

### 8.1 卡片层级约束

- **定义**：具有卡片样式的容器（`bg-panel` + `shadow` + `rounded`）的**嵌套深度**。
- **规则**：**卡片层级最多 2 层**。
- **目的**：Chrome Extension 窗口小，过多层级会导致视觉拥挤和空间浪费。

---

## 九、强制约束（写进 Cursor Rules）

1. 禁止在组件中直接写 Arbitrary Values (如 `p-[13px]`)。
2. 所有样式必须来自 `tailwind.config.ts` 定义的 Utility Classes。
3. 不允许新增 Token，除非明确说明理由。
4. 业务组件不允许写复杂样式，应复用 `ui/` 组件。
5. 卡片层级最多 2 层。

---

## 十、修改策略速查表

| 想改什么 | 改哪里 |
|----------|--------|
| 页面更松/更紧 | `tailwind.config.ts` -> `spacing.page / section` |
| 卡片太挤 | `tailwind.config.ts` -> `spacing.card` |
| 卡片嵌套过深 | 调整组件结构，扁平化 |
| 不够高级 | `tailwind.config.ts` -> `boxShadow / borderRadius` |
| 信息不清晰 | `tailwind.config.ts` -> `fontSize / colors.text` |
| 风格偏了 | `tailwind.config.ts` (Theme) |

---

## 十一、一句总结

- **React** 负责结构与逻辑。
- **Tailwind Config** 负责审美与一致。
- **你** 负责裁决改 Config 还是改结构。
